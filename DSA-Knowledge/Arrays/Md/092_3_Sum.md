# 3 Sum

## Difficulty
Medium

---

## Pattern
Two Pointers / Sorting

---

## Recognition Clues
- Find all unique triplets in an array that sum to 0.\n- The solution set must not contain duplicate triplets.

---

## Core Insight
- Sort the array first to allow Two-Pointer search and easy duplicate skipping.\n- Fix the first element `nums[i]`. Search for the remaining target `-nums[i]` in `nums[i+1...N-1]` using left and right pointers.\n- Skip duplicates of `nums[i]`, `nums[left]`, and `nums[right]` to ensure unique triplets.

---

## Interview Explanation
1. **Brute Force Idea**: Check all triplets using three nested loops. If they sum to 0, sort the triplet and add to a set to remove duplicates.
2. **Why Inefficient**: Checking all triplets takes $O(N^3)$ time.
3. **Key Observation**: If we sort the array and fix the first element, we reduce the problem to Two Sum on a sorted array, which can be solved in $O(N)$ time with two pointers. This brings complexity down to $O(N^2)$.
4. **Optimal Solution Intuition**:
   - Sort the array.
   - Loop `i` from 0 to $N-1$. If `nums[i] > 0`, break early (since positive numbers cannot sum to 0). If `i > 0` and `nums[i] == nums[i-1]`, skip to avoid duplicates.
   - Set `left = i + 1` and `right = N - 1`.
   - While `left < right`:
     - Calculate `curr_sum = nums[i] + nums[left] + nums[right]`.
     - If `curr_sum == 0`, append the triplet. Advance `left` and decrement `right`, skipping duplicate values of both.
     - If `curr_sum < 0`, increment `left`. Else, decrement `right`.
5. **Time Complexity**: $O(N^2)$ due to the nested pointer search.
6. **Space Complexity**: $O(1)$ auxiliary space (ignoring sorting overhead).

---

## Brute Force
- **Idea**: Check all possible triplets and use a set to filter duplicates.
- **Code**:
```python
def threeSum_brute(nums):
    ans = set()
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    triplet = tuple(sorted([nums[i], nums[j], nums[k]]))
                    ans.add(triplet)
    return [list(x) for x in ans]
```
- **TC**: $O(N^3)$
- **SC**: $O(N)$

---

## Optimal Approach
- **Algorithm**: Sort followed by fixed-index two-pointer search.
- **Why it works**: Sorting enables a two-pointer scan which reduces the search of the remaining pair from $O(N^2)$ to $O(N)$, and allows skipping identical elements in-place.
- **Complexity**: Time: $O(N^2)$, Space: $O(1)$ auxiliary space.

---

## Optimal Code
```python
def threeSum(nums):
    nums.sort()
    ans = []
    n = len(nums)
    
    for i in range(n):
        # Since array is sorted, if nums[i] > 0, no positive numbers can sum to 0
        if nums[i] > 0:
            break
            
        # Skip duplicate values for the first element
        if i > 0 and nums[i] == nums[i-1]:
            continue
            
        left, right = i + 1, n - 1
        while left < right:
            curr_sum = nums[i] + nums[left] + nums[right]
            
            if curr_sum == 0:
                ans.append([nums[i], nums[left], nums[right]])
                
                # Skip duplicate elements for left and right pointers
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                    
                left += 1
                right -= 1
            elif curr_sum < 0:
                left += 1
            else:
                right -= 1
                
    return ans
```

## Test Cases
- **Normal Case**: Input: `nums = [-1, 0, 1, 2, -1, -4]`\n- Output: `[[-1, -1, 2], [-1, 0, 1]]`
- **Hard Case**: Input: `nums = [0, 0, 0, 0]`\n- Output: `[[0, 0, 0]]`
- **Edge Case**: Input: `nums = []`\n- Output: `[]`

---

## Common Mistakes
- Forgetting to skip duplicates of the first element `nums[i]`.\n- Forgetting to skip duplicates of `nums[left]` and `nums[right]` after finding a valid triplet, leading to duplicate triplets in output.

---

## Killer Edge Cases
- All elements are zero.\n- No triplets exist.\n- Fewer than 3 elements in the array.
