# 4 Sum

## Difficulty
Medium

---

## Pattern
Two Pointers / Sorting

---

## Recognition Clues
- Find all unique quadruplets in an array that sum to a target value.\n- The solution set must not contain duplicate quadruplets.

---

## Core Insight
- Sort the array to enable Two-Pointer scan and duplicate skipping.\n- Fix the first two elements `nums[i]` and `nums[j]` using two nested loops.\n- Search for the remaining target `target - nums[i] - nums[j]` in `nums[j+1...N-1]` using left and right pointers.

---

## Interview Explanation
1. **Brute Force Idea**: Check all quadruplets using four nested loops. If they sum to target, sort them and insert into a set.
2. **Why Inefficient**: Takes $O(N^4)$ time.
3. **Key Observation**: By sorting the array and fixing the first two elements, we reduce the problem to Two Sum, which takes $O(N)$ with two pointers. This brings the total time complexity down to $O(N^3)$.
4. **Optimal Solution Intuition**:
   - Sort the array.
   - Nest loops `i` from 0 to $N-1$ and `j` from `i + 1` to $N-1$. Skip duplicates for both loops.
   - Set `left = j + 1` and `right = N - 1`.
   - While `left < right`:
     - Calculate `curr_sum = nums[i] + nums[j] + nums[left] + nums[right]`.
     - If `curr_sum == target`, append the quadruplet and shift pointers, skipping duplicates. If too small, `left += 1`. If too large, `right -= 1`.
5. **Time Complexity**: $O(N^3)$ due to three nested search dimensions.
6. **Space Complexity**: $O(1)$ auxiliary space.

---

## Brute Force
- **Idea**: Check all possible quadruplets and filter duplicates.
- **Code**:
```python
def fourSum_brute(nums, target):
    ans = set()
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                for l in range(k + 1, n):
                    if nums[i] + nums[j] + nums[k] + nums[l] == target:
                        quad = tuple(sorted([nums[i], nums[j], nums[k], nums[l]]))
                        ans.add(quad)
    return [list(x) for x in ans]
```
- **TC**: $O(N^4)$
- **SC**: $O(N)$

---

## Optimal Approach
- **Algorithm**: Double-fixed pointer sorting search.
- **Why it works**: Fixing two boundaries reduces the search space for the remaining pair to a linear two-pointer scan, saving two dimensions of nested loops.
- **Complexity**: Time: $O(N^3)$, Space: $O(1)$ auxiliary space.

---

## Optimal Code
```python
def fourSum(nums, target):
    nums.sort()
    ans = []
    n = len(nums)
    
    for i in range(n):
        # Skip duplicate values for the first element
        if i > 0 and nums[i] == nums[i-1]:
            continue
            
        for j in range(i + 1, n):
            # Skip duplicate values for the second element
            if j > i + 1 and nums[j] == nums[j-1]:
                continue
                
            left, right = j + 1, n - 1
            while left < right:
                curr_sum = nums[i] + nums[j] + nums[left] + nums[right]
                
                if curr_sum == target:
                    ans.append([nums[i], nums[j], nums[left], nums[right]])
                    
                    # Skip duplicate values for left and right pointers
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                        
                    left += 1
                    right -= 1
                elif curr_sum < target:
                    left += 1
                else:
                    right -= 1
                    
    return ans
```

## Test Cases
- **Normal Case**: Input: `nums = [1, 0, -1, 0, -2, 2]`, `target = 0`\n- Output: `[[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]`
- **Hard Case**: Input: `nums = [2, 2, 2, 2, 2]`, `target = 8`\n- Output: `[[2, 2, 2, 2]]`
- **Edge Case**: Input: `nums = []`, `target = 0`\n- Output: `[]`

---

## Common Mistakes
- Forgetting to skip duplicates for the outer and inner loops (`i` and `j`).\n- Incorrect pointer initialization (e.g. `left = j` instead of `j + 1`).

---

## Killer Edge Cases
- Fewer than 4 elements.\n- All elements are the same.\n- Target sum is negative.
